# V29 Research — Edge 衰退警報（回測驗證）

**研究日期**：2026-07-03
**動機**：V14 是 regime-dependent（G8 時序翻轉 FAIL），edge 依賴 ETH 的壓縮突破延續結構，
會過期且沒有先行指標。單筆風險有 SafeNet、回撤有熔斷，**唯一沒被管住的是
「市場結構安靜地變了」** —— 需要一個規則化的衰退警報，把「發現 edge 死亡」從憑感覺變成有規則。
**結論**：**PROMOTED — 雙層 CUSUM 警報**（黃燈 h=600 凍結加碼 / 紅燈 h=800 退回 200U 地板），
歷史兩年零誤觸紅燈（峰值 559）、edge 反轉 1.1 個月偵測、歸零 2.8 個月、漏報 2%。

---

## 方法論

歷史兩年 edge 一直是活的 → **誤報率**用真實序列 + 月塊 bootstrap 驗證；
**偵測力**用合成死亡情境驗證：

| 情境 | 定義（死亡後每筆 PnL） |
|---|---|
| A 歸零 | 歷史分佈抽樣去均值（E=0，突破不再延續，只剩費用摩擦） |
| B 反轉 | 均值鏡像為負（-μ，結構反噬，= G8 情境） |
| C 緩衰 | 6 個月線性從 μ 衰到 0 後停住（最難測的溫水煮蛙） |

所有指標以 **200U 基準 R 單位**計算（實盤實作：`pnl_usd × 200/當筆保證金`），
不受複利/保證金調整影響。單筆 μ=$28.6、σ=$69.9（realistic、273 筆）。

**候選規則**：CUSUM 下移變點（`S_t = max(0, S_{t-1} + (μ/2 − x_t))`，S>h 觸發）、
滾動 W 筆 PnL 和、滾動 TP 出場佔比。

  —— h=600 黃燈在那段會亮，回頭看那正是該凍結加碼的時期，行為合理

## 採用方案：雙層 CUSUM（R 單位，k=μ/2=14.3）

```
每筆交易出場後更新：S = max(0, S + (14.3 − pnl_R))，pnl_R = pnl_usd × 200/保證金

🟡 黃燈 S > 600：凍結複利加碼（保證金停在當前值，不再上調）
   歷史會在 2026-01 類疲軟期亮起；誤報成本低（只是暫停加碼）
🔴 紅燈 S > 800：保證金退回 200U 地板 + 人工檢視策略是否失效
   歷史兩年零觸發；edge 反轉 ~1.1 個月內攔截、歸零 ~2.8 個月、漏報 2%
   誤報率 24%/2年（≈ 每 8 年一次白白降槓桿，成本可接受）

恢復條件（設計約定，未深度回測）：紅燈後 S 歸零、以 200U 續跑觀察，
滾動 40 筆累計 > 0 才恢復保證金 schedule。
```

**經濟量化**：最壞情境 B（edge 反轉）偵測前失血 ≈ 1.1 月 × 月虧熔斷上限（R225/月）
≈ R250~350（300U 下 ~$400~500）—— 警報 + 既有熔斷把「edge 死亡」的總代價封在
一個月虧損量級，而不是無限期流血。
